# とけいマスター 仕様書

小学1年生にアナログ時計の読み方を教えるためのフロントエンド完結型 Web アプリ。

## 目的

- 子どもがアナログ時計をドラッグ操作して時間を選ぶ。
- 選んだ時間の読み方を画面表示と音声で確認できる。
- 大人が「何時何分に何をする」というスケジュールを保存し、子どもに教える。

## 動作環境

- Android Chrome / iOS Safari / 最新のデスクトップブラウザ
- フロントエンドのみ（バックエンド無し）。データは `localStorage` に保存。
- PWA としてホーム画面にインストール可能。

## 画面構成

タブで切り替える 2 画面。

| タブ | 役割 |
| --- | --- |
| とけい | アナログ時計を操作して時刻を選び、読み方を確認・保存する |
| スケジュール | 保存した時刻＋テキストをタイル一覧として表示・並び替え・削除する |

## 時計画面

### アナログ時計

- 文字盤は算用数字（1〜12）。
- 60 分の目盛り（5 分ごとに強調）を表示。
- 長針（分針, ピンク）と短針（時針, 黒）を表示。
- 中心キャップを表示。
- 長針・短針はそれぞれドラッグ可能。
  - 長針ドラッグ
    - **1 分ごとにスナップ**。
    - 短針は分の進行に追従して滑らかに移動。
    - 長針が 0 分を跨ぐと短針が 1 時間進む（後退も同様に巻き戻る）。
    - ドラッグ中、文字盤の `1〜12` の数字は **`5,10,…,55,0` の分数字に切り替わる**。
  - 短針ドラッグ
    - 1 時間単位でスナップ。長針は 0 分にリセット。
- ポインタイベントベースで実装（マウス/タッチ/Apple Pencil 共通）。

### 表示

- 大きな「H時M分」表示。各漢字・数字に **ふりがな（ruby）** を付ける。
- 助数詞の音便を網羅:
  - 1 時 → いちじ / 4 時 → よじ / 7 時 → しちじ / 9 時 → くじ
  - 1 分 → いっぷん / 3 分 → さんぷん / 6 分 → ろっぷん / 8 分 → はっぷん
  - 10 分 → **じっぷん**（仕様）
  - 0 分 → ぜろふん（学習用にゼロ表記の読みを表示）
  - 21 分 → にじゅういっぷん 等

### 操作ボタン

すべてアイコン＋テキスト。

| ボタン | 役割 |
| --- | --- |
| 🔊 にほんごでよむ | 表示中の時刻を日本語で読み上げ。時と分の間に間を入れる |
| 🔊 Read in English | 表示中の時刻を英語で読み上げ。`o'clock` / `oh X` ルール |
| ⏱ いまのじかん | 端末の現在時刻に合わせる |
| 💾 スケジュールにほぞん | テキスト入力フォームを開き、現在の時計＋テキストを保存 |

#### 英語読み上げルール

- 1:00 → `one o'clock`
- 1:01〜1:09 → `one oh one` … `one oh nine`
- 1:10〜1:59 → `one ten` … `one fifty nine`
- 0/24 時は 12 時として扱う。

## スケジュール画面

- 保存済みスケジュールをタイルでグリッド表示。
- 各タイルに以下を表示:
  - 静的アナログ時計（針が読めるサイズ）
  - 「H時M分」+ ふりがな
  - 大人が入力したテキスト
  - 削除ボタン（✕）
- **タップで読み上げ**（日本語）。
- **ドラッグ&ドロップで並び替え**。デスクトップは HTML5 DnD、タブレット/スマホは長押し（350ms）で reorder モード。
- 並び順は localStorage に永続化。
- 編集機能は無し（削除→再登録）。

## データ仕様

`localStorage` キー: `tokei-master.schedules.v1`

```ts
interface Schedule {
  id: string;        // 一意ID（タイムスタンプ + ランダム）
  hour: number;      // 1..12
  minute: number;    // 0..59
  text: string;      // 大人が入力したテキスト
  createdAt: number; // UNIX ms
}
type Stored = Schedule[]; // 並び順そのまま保存
```

## UI / UX

- 基準デバイス: タブレット横持ち。レスポンシブで縦持ち・スマホ・PC にも追従。
- 子ども向けのフォント・カラー（パステル系、太字、コントラスト確保）。
- すべての操作要素にアイコンを付与。
- ボタンの最小タップサイズは 56px。
- ドラッグ中はカーソル/ホバー演出を変更し、操作中であることを明示。

## アクセシビリティ

- ボタンに `aria-label`。タブに `role="tab"` と `aria-selected`。
- ふりがなは `<ruby>` を使用、スクリーンリーダー互換。
- `user-scalable=no` を指定しているため、初学者がピンチで誤操作しない。読みづらい場合は端末側のフォントサイズで対応する想定。

## PWA

- `public/manifest.webmanifest` に名前・色・アイコン情報。
- `public/sw.js` で stale-while-revalidate なオフラインキャッシュ。
- `display: standalone`、起動 URL は `./`、scope は `./`。
- アイコンは SVG マスターから 192/512/512(maskable) を `scripts/build-icons.mjs` で生成。

## デプロイ

GitHub Actions で `main` ブランチへの push を契機に Vite ビルドし、GitHub Pages にデプロイする。プロジェクトページとして配信するため `VITE_BASE` でリポジトリ名サブパスを注入する。

## 既知の制限

- Web Speech API の音声品質は端末/ OS により差がある（特に iOS Safari の英語/日本語ボイス）。
- iOS Safari ではユーザー操作直後でないと `speechSynthesis.speak()` がブロックされるため、読み上げはボタン操作時のみ実行する。
- `localStorage` のため複数端末間の同期はしない。
